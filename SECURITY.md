# Security Policy

## Supported Versions

Ryxen is currently in the `1.x` line. Security fixes are applied to the latest
release on `main` and the latest production deployment.

| Version | Supported |
| ------- | --------- |
| 1.x     | Yes       |
| < 1.0   | No        |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
pull requests, or discussion threads.

Report vulnerabilities privately by email to:

- `nikolas.magnet177@gmail.com`

Include as much detail as possible:

- affected environment or URL
- steps to reproduce
- proof of concept, logs, screenshots, or request samples
- impact assessment
- any suggested remediation

What to expect after reporting:

- acknowledgment within 72 hours
- an initial triage update within 7 days
- follow-up updates as the investigation progresses

If the report is accepted, we will work on a fix, validate it, and coordinate
disclosure when appropriate. If the report is declined, we will explain why,
for example if it is out of scope, not reproducible, or not a security issue.

Please avoid testing that could degrade service, access other users' data, or
disrupt production systems.
