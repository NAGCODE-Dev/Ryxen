## Confirmed issues

### 1) Running and Strength import a missing symbol
Both `sports/running/main.js` and `sports/strength/main.js` import `getAthleteDashboard` from `src/core/services/gymService.js`, but `gymService.js` does not export that function yet.

**Impact:** both sport-specific entrypoints can fail during module evaluation, preventing those screens from loading.

**Suggested fix:** add:

```js
export async function getAthleteDashboard(params = {}) {
  const search = new URLSearchParams();
  if (params?.sportType) search.set('sportType', params.sportType);
  const suffix = search.toString() ? `?${search.toString()}` : '';
  return apiRequest(`/athletes/me/dashboard${suffix}`, { method: 'GET' });
}
```

### 2) Service worker cache list is out of date
`sw.js` precaches legacy assets such as `src/app.js`, but does not precache newer live entrypoints like `src/hub/main.js`, `src/hub/styles.css`, `sports/running/main.js`, or `sports/strength/main.js`.

**Impact:** stale caches, broken first-load/offline behavior, and mismatches between current navigation and cached assets.

**Suggested fix:** update `CORE_ASSETS`, remove dead entries, and bump the cache version.

### 3) PWA cache list mixes hub/app generations
The current cache list contains hub, legacy cross app, and partial new sport flows at the same time.

**Impact:** harder invalidation and higher chance of serving wrong assets after deploy.

**Suggested fix:** keep only real, current entrypoints in `CORE_ASSETS`.
